# Hotel POS SOP (One-Page Guide)

This quick guide is for front desk, waiters, supervisors, and managers using the current Hotel POS module.

## 1) Start of Shift

- Open `Sales -> Hotel POS`.
- Confirm internet status (`Online`/`Offline`) at the top.
- Select your table and click `Open` in the session bar.
- Confirm your role access (waiter/supervisor/manager/accountant).

## 2) Table and Session Use

- Use `Table Layout` to choose tables (`T1`, `VIP1`, etc.).
- Status color guide:
  - Green: available
  - Amber: reserved
  - Blue: cleaning
  - Red: occupied
- Update table status and waiter assignment from the selected table section.
- Close table session only after bill is cleared.

## 3) Taking Orders

- Choose `Department` and `Menu` (Breakfast/Lunch/Bar/Room Service).
- Search item in `Search products`.
- Click product tile to add to cart.
- For each cart item:
  - Adjust quantity with `+`/`-`.
  - Add comment/note where needed.
  - Set `Course` and `Fire timing` if required.
  - Use `Modifier` for custom requests.

## 4) Customer and Pricing

- Set `Table` and (optional) `Customer`.
- Use `Promo code` / `Promo %` if approved.
- Mark VIP only for approved VIP guest records.
- Verify total before posting.

## 5) Posting Order Actions

- `Send to Kitchen`: sends order for preparation, no payment posted.
- `Pay Now`: posts completed payment immediately (select payment method first).
- `Bill to Room`: posts charge to selected active room stay.
- `Credit Sale`: records sale with pending payment.
- `Hold Ticket`: parks current cart for later resume.

## 6) Printing Bill

- Ensure cart has correct items and totals.
- Click `Print Bill` in Hotel POS action buttons.
- In preview popup, click `Print`.
- Hand printed copy to guest/customer.

## 7) Order Queue Management

- Use queue filters: `Status` and `Station` (Kitchen/Bar/Dessert).
- Click `Edit` to adjust queued order date/items if needed.
- Save changes immediately after edits.

## 8) Posted Transactions and Controls

- `Posted Hotel POS Transactions` shows payments for selected date.
- Editable by authorized roles (manager/supervisor/accountant/admin).
- Refunded transactions are locked.
- Always enter valid void/refund reason where prompted.
- Manager PIN override is required for sensitive waiter actions.

## 9) End of Shift Checklist

- Ensure no active cart is left unsaved.
- Confirm all prepared orders are posted correctly.
- Reconcile cash/mobile/card totals with posted payments.
- Clear or close table sessions that are complete.
- Report any mismatch immediately to supervisor.

## 10) Do/Don’t Rules

### Do
- Confirm table, items, and totals before posting.
- Use comments/modifiers for special guest requests.
- Print and provide bill after verification.
- Use role-approved workflows for edits/refunds.

### Don’t
- Do not post duplicate payments for the same order.
- Do not refund/void without reason and approval.
- Do not close a session with pending unsettled orders.
- Do not share manager PIN.
